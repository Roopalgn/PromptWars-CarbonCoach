## New Way to Build

In PromptWars: Virtual, we are moving away from manual coding and moving toward intent-driven development. Join us for a bi-weekly virtual hackathon where you'll utilize **Google Antigravity** to build functional, real-world applications in a fraction of the time.

## How It Works: The 14-Day Cycle

Every challenge follows these four phases:

### Phase 1: Challenge Release
- New challenge released every two weeks (on Monday)
- Real-world industry problems designed for all skill levels: Beginner, Intermediate, and Advanced

### Phase 2: Building Phase
- Use your creativity and vibe coding skills
- You must use **Google Antigravity** to build a functional app

### Phase 3: Submission Phase (By Day 13)
Submit your solution in two parts:
- **Technical**: Your code and a live preview of your app
- **Narrative**: A technical blog post and a LinkedIn post sharing your "Build-in-Public" journey

### Phase 4: Evaluation & Announcement (Day 14)
- Our experts will review your app
- If your submission is valid, you will move up the leaderboard and earn your Prompt Credits

## Frequently Asked Questions

### What is a "Dual Submission"?
To qualify, you must submit your project (the code) and a narrative (a blog post or article). We want to see how you solved the problem, not just the final result.

### Do I have to use Google Antigravity?
Yes. To keep the competition fair and focused, using Google Antigravity is a requirement for all submissions.

### What happens if I miss a week?
No problem! You can jump in at any time. However, participating every week helps you climb the cumulative leaderboard for bigger rewards.

### How are participants selected?
The PromptWars: Virtual team reserves the right to invite, shortlist, and confirm all participants. To maintain the integrity of the competition, our selection criteria remain confidential and are managed at the sole discretion of the organizing committee.

## Prompt Credits

Prompt Credits are the reward currency of PromptWars: Virtual. Every bi-weekly challenge, the top 400 participants on the leaderboard earn Prompt Credits. The higher you finish, the more you earn.

- Credits accumulate across multiple challenges and **never expire** (until Jan 1, 2027)
- Even if you only join a few challenges, your credits keep adding up

### Key Things to Remember

- **Expiration**: Prompt Credits expire on **1st Jan 2027** — earn across any of the 18 challenges, but redeem before this date
- **Credits stack up**: Every challenge you join adds to your total. A Rank 50 player who joins 12 challenges earns 5,100 credits — enough for a Backpack

- **Only top 400 earn credits**: Finishing outside the top 400 means no credits for that round — but previously earned credits are always safe
- **Spending flexibility**: You do not need to spend all your credits at once; save them up and buy the item you actually want

### AI Code Analysis Criteria
Your code submission is evaluated by AI on the following criteria:

- **Code Quality** — How clean, readable, and well-structured the code is
- **Security** — Whether the code follows safe practices and avoids common vulnerabilities
- **Efficiency** — How well the code utilizes resources like time and memory
- **Testing** — How easily the code can be tested, validated, and maintained over time
- **Accessibility** — How usable the solution is for diverse users and environments
- **Google Services** — How effectively Google Services are integrated into the solution

> The AI analysis typically takes 4–5 minutes.

## Submission Instructions

### 1. Before You Begin
Make sure the following prerequisites are completed:

- Google Antigravity is downloaded and set up on your system
- Git is installed and configured
- You have an active GitHub account
- You are able to create and manage public repositories

### 2. Important Rules

- You have a maximum of **3 submission attempts**
- The repository size must be **less than 10 MB**
- The GitHub repository must be **public**
- The repository should contain **only one branch**
- Failure to follow these rules may result in your submission not being evaluated

### 3. Challenge Expectations
Your solution should demonstrate:

- Ability to build a smart, dynamic assistant
- Logical decision making based on user context
- Effective use of Google Services
- Practical and real-world usability
- Clean and maintainable code

> Participants must choose one of the provided challenge verticals and design their solution around that persona and logic.

### 4. How to Work on Your Project

- Create a new repository on GitHub
- Ensure the repository is set to **public**
- Open Google Antigravity
- Clone your repository inside Antigravity
- Build your solution through prompting and coding
- Regularly commit and push your progress
- Keep all work within a **single branch**

### 5. What to Submit

Your submission must include:

- A public GitHub repository link
- Complete project code inside the repository
- A `README` explaining:
  - Your chosen vertical
  - Approach and logic
  - How the solution works
  - Any assumptions made

### 6. Evaluation Focus Areas
Submissions will be reviewed on:

- **Code Quality** — Structure, readability, maintainability
- **Security** — Safe and responsible implementation
- **Efficiency** — Optimal use of resources
- **Testing** — Validation of functionality
- **Accessibility** — Inclusive and usable design
- **Google Services** — Meaningful integration of Google Services
